## 编码规范

- **语言**：返回中文回复
- **代码风格**：只返回修改的代码，做最小改动，不炫技
- **响应式**：所有页面必须同时在 H5（移动端）和 PC 上可用
- **工具函数**：通用函数放在 `src/utils/`，优先使用现有工具
- **HTTP**：所有 HTTP 请求使用 axios
- **文档**：功能变更时更新工具描述
- **SEO**：所有工具页面需要详细描述
- **删除**：删除 UI 元素时，同时删除相关方法和变量

## 核心原则（权重）

1. **体验性（50%）**：美观的 UI、流畅的交互、细腻的动画
2. **完整性（30%）**：功能完整、流程合理、PC/H5 体验一致
3. **技术性（20%）**：架构清晰、性能优化、可维护性

## UI 指南

- 适当使用毛玻璃效果、渐变动画、淡入效果
- 保持极致简洁和高级的 UI 美学
- 利用 Tailwind CSS 类 + Element Plus 组件
